# API利用規約ドラフト（外部開発者向け）

## 1. 適用範囲

- **対象者**: STARLISTと契約した外部企業パートナー、開発者、リサーチ会社。  
- **対象API**: `STAR_STATS_PUBLIC`, `MARKET_INSIGHTS`, `SUPER_CHAT_SUMMARY` など、スター統計や匿名集計を提供するすべての外部公開エンドポイント。  
- **前提**: 利用にはデータライセンス契約と API キーの発行が必要。  

## 2. 利用可能な用途

- 自社内の分析やダッシュボード表示（自社サービス内で完結）。  
- スター本人または代理店との契約に基づくビジネス活動（別途同意を得たスターに限る）。  
- STARLISTの公認パートナーとしてのマーケットレポートやアフィリエイト向け集計。  
- 提供データを二次加工する際も、「匿名化・集計」を維持し、PIIを復元可能な形にしない。  

## 3. 禁止事項

- 再販売・外部再配布（例: APIの生データをそのまま販売、他社に渡す）。  
- 再識別行為（数的/命名情報から個人を特定する試み）。  
- オープンデータ化・Creative Commons 付与。  
- 誹謗中傷・ヘイト・個人攻撃用途。  
- STARLISTロゴや名称を許可なく商品化。  

## 4. レート制限とSLA

- 標準キー: 1分あたり 60 req 上限。プレミアムパートナーは事前合意の上で倍増する。  
- 99%のリクエストが 500ms 以内に応答することを目標とするが、バースト時は 429 を返しリトライを促す。  
- SLA違反時は次回請求時にクレジットを付与する代わりに、同等処理を速やかに実施。  

## 5. 責任と違反対応

- APIレスポンスに含まれる誤りやアウトプットのビジネス損害のみをSTARLISTが補償（その他の間接損害は限定）。  
- 再販売などの重大違反が判明した場合は即時 API キー停止＋契約解除。  
- 違反の記録は `ops_metrics` に残し、法務・PMへ報告。  

## 6. データライセンスの順守

- 本規約は `DATA_LICENSE_AND_EXTERNAL_API_POLICY.md` の原則（PII禁止、集計/統計優先、再識別禁止）を前提とする。  
- 新規APIを追加する場合は、必ず「DATA LICENSE → 匿名化ガイド → API Terms」の順で韻を踏むレビューを経ること。

# API利用規約ドラフト（外部開発者向け）

## 1. ドキュメントの位置づけ

この章は、STARLIST が外部企業・開発者に対して API 形式で統計/集計データを提供する際の利用規約案です。契約書とエンドユーザーライセンスのリファレンスとして用い、データライセンス憲法 (`DATA_LICENSE_AND_EXTERNAL_API_POLICY.md`) を具体的な条項に落とし込む役割を果たします。

## 2. 対象と範囲

- **対象API**: STARLIST が公開する統計系エンドポイント（例: `/api/stars/{starId}/dashboard`, `/api/market-insights`）
- **対象者**: 外部法人、外部エンジニア、データパートナー（本規約の受諾者）
- **提供データ**: 集計/統計ベースのメトリクス、グロス売上、スパチャ傾向など。PIIや生ログを含まないことを前提。

## 3. 利用条件

### 3.1 権利と免許

- STARLIST は本規約に基づき、利用者に対して**非独占・非移譲・再販不可**の利用権を付与する。
- 利用権は契約期間中に限り有効で、APIキーと利用者IDによって管理される。
- 利用者はSTARLISTのブランド表記（`STARLISTデータライセンス`）をすべての外部公開物に明示する必要がある。

### 3.2 利用目的

- 許容される目的: 自社内分析、ダッシュボード表示、代理店が契約スターの分析業務、限定した決済トリガーなど。
- 不可: データの再販売、再配布、再識別（Re-identification）、AI生成物への無断統合、誹謗中傷の支援など。

### 3.3 レート制限と料金

- APIはトークン毎に1分あたり/日次クォータを設け、基本プラン・プレミアムプランで上限が変わる。
- 料金体系は月額 + 利用量（スロット数×リクエスト数）で構成し、超過時は一定額を課金する。

## 4. 利用者責任

- APIキーの管理責任を負い、共有・流出を認めない。
- 監査ログ・アクセスログの保存を同意し、異常アクセスが確認された場合は直ちにSTARLISTに通報。
- データの二次加工時には「STARLISTデータライセンス」表記と参考URLを併記する。

## 5. 利用停止・解除

- 再販・再識別・PIIの提供など本規約違反が見つかった場合：即時APIキー停止、利用者への通知、契約解除。
- 利用停止中のデータは一切再利用できず、再開には再契約 + 監査レビューが必要。
- 緊急対応として、STARLISTはAPIリクエストの一時的レート制限強化やインフラ遮断（WAFルール追加）を実施できる。

## 6. 補足

- STARLISTは当規約を所管する内部チーム（PM/法務）を指定し、定期的なレビューを行う。
- 重要な変更時には、`version` と `updated` を更新し、PRレビューにて「開発」「PM」「法務（弁護士）」の三層で確認する。
