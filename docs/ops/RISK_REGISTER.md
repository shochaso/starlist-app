# STARLIST リスクレジスター

## 背景

STARLISTプロジェクトにおける潜在的なリスクを体系的に特定・評価・管理するための公式ドキュメントです。本レジスターはプロジェクトの成功確率を最大化し、予期せぬ問題発生時の迅速な対応を可能にします。

## 要件

### 機能要件
- [x] リスクの体系的特定と分類
- [x] 影響度・発生確率の定量評価
- [x] 対策計画の策定と責任者割り当て
- [x] リスクの継続監視とステータス更新

### 非機能要件
- [x] リアルタイムでのリスク評価更新
- [x] 全ステークホルダーの参照可能
- [x] 定期的な見直し（週次）

## リスク評価基準

### 影響度 (Impact)
1. **高 (重大)**: プロジェクトの中止や大幅な遅延を招く
2. **中 (中程度)**: スケジュールの遅延やコスト超過を招く
3. **低 (軽微)**: 一時的な作業効率低下にとどまる

### 発生確率 (Probability)
1. **高 (70%以上)**: ほぼ確実に発生する
2. **中 (30-70%)**: 発生する可能性が高い
3. **低 (30%以下)**: 発生する可能性が低い

### リスク優先度 = 影響度 × 発生確率

#### 6. Markdown CIガバナンス未整備
**カテゴリ**: プロセスリスク
**説明**: Markdown検証・リンクチェックが抱える既知のフォーマット問題（テーブルのカラムズミス、言語指定のないコードフェンスなど）と、`npm run lint:md` が Node 20 の ES モジュールスクリプトを必要とする点により、CI によるドキュメント検証が一貫して失敗している

**影響度**: 中 (リリースゲートが通らず、ドキュメント変更のデプロイに支障)
**発生確率**: 高 (既に 2,400 件超のエラー/警告が `docs/ops/MD_CI_REPORT.md` に蓄積)
**優先度**: 8

**現在のステータス**: 🔴 Blocked
**責任者**: Documentation Team
**対策計画**:
- `docs/ops/MD_CI_REPORT.md` に Markdown CI の実行結果を記録し、table-columns / codefence-language など現行エラーを明示
- テンプレート優先の `docs/ops/*` を対象に、テーブル・コードブロックのフォーマットを段階的に修正
- Node 20 環境が整うまで、shim や Node 18 でも動作できる workaround の検討
- CI の失敗をドキュメントし、再発防止のため監視・共有ルールを整備

**残存リスク**: 中
**最終更新**: 2025-11-17

## リスク管理プロセス

### 1. リスク識別フェーズ
- **継続的監視**: デイリースクラムでのリスク共有
- **定期レビュー**: 週次リスク評価会議
- **ステークホルダー入力**: チームからのリスク報告

### 2. リスク評価フェーズ
- **影響度評価**: プロジェクト全体への影響分析
- **発生確率評価**: 過去データと専門家判断による
- **優先順位付け**: 影響度×発生確率によるスコアリング

### 3. リスク対策フェーズ
- **対策計画策定**: 具体的なアクションと責任者の設定
- **リソース配分**: 対策実施に必要なリソース確保
- **タイムライン設定**: 対策完了目標の設定

### 4. リスク監視フェーズ
- **ステータス追跡**: 定期的なリスク状態更新
- **効果測定**: 対策の有効性評価
- **再評価**: 状況変化に応じたリスク再評価

## DoD (Definition of Done)

### リスク管理完了の条件
- [x] 全リスクの識別と評価完了
- [x] 対策計画の策定と責任者割り当て
- [x] 監視プロセスの確立
- [x] ステークホルダーのリスク認識統一
- [x] 定期見直しメカニズムの確立

## Runbook

### 新規リスク登録プロセス
1. **リスク検知**: 新しいリスクの特定
2. **情報収集**: 影響度・確率の評価材料収集
3. **評価実施**: 定量評価と優先順位付け
4. **対策策定**: 具体的な対応計画作成
5. **責任者設定**: 対策実行責任者の任命

### 緊急リスク対応プロセス
1. **緊急度判定**: 即時対応の必要性評価
2. **影響最小化**: 暫定的な対策実施
3. **詳細分析**: 根本原因と長期対策の検討
4. **報告**: ステークホルダーへの影響報告

## リスク指標

### 月次レポート指標
- **リスク数**: 現在登録中のリスク総数
- **高優先度リスク**: 優先度8以上のリスク数
- **未対策リスク**: 対策未完了のリスク数
- **リスク軽減率**: 対策により軽減されたリスクの割合

### 四半期レビュー指標
- **新規リスク発生率**: 新規リスクの発生頻度
- **対策完了率**: 計画された対策の完了率
- **影響回避額**: リスク対策により回避した損失額
