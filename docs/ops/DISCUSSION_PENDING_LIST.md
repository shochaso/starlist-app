## 議論テンプレート

## Runbook

### 議論管理プロセス
1. **識別**: 新しい議論事項の発見・登録
2. **評価**: 緊急度・影響度の評価
3. **議論**: 関係者会議での検討
4. **決定**: 合意形成と文書化
5. **実施**: 決定に基づく実装
6. **検証**: 実施結果の確認

### 定期レビュープロセス
- **日次**: 新規項目の確認と登録
- **週次**: 進行中の項目の進捗確認
- **月次**: 全項目の見直しと優先順位調整

## リスク

### プロセス的リスク
- **議論の停滞**: 関係者の合意形成が遅延
  - **対策**: ファシリテーターの配置と期限設定

### 技術的リスク
- **前提条件の変化**: 議論中に技術環境の変化
  - **対策**: 定期的な前提条件の見直し
